**Objective section:**
Describe what the command does: reads the user's profile, detects active countries, provides personalized investment tax guidance for Germany and/or Brazil, calculates Sparerpauschbetrag usage, walks through Vorabpauschale, and surfaces DBA interaction for cross-border users.

Creates: advisory output in the conversation (no file output — this is a conversational advisor).

**Process phases:**

**Phase 1: Validation**
- Profile gate: `[ -f .finyx/profile.json ] || { echo "ERROR: No financial profile found. Run /finyx:profile first."; exit 1; }`
- Read `.finyx/profile.json`
- Detect active countries:
  - Germany active if: `countries.germany.tax_class != null`
  - Brazil active if: `countries.brazil.ir_regime != null`
  - Cross-border if: `identity.cross_border == true`
- If neither country active: "ERROR: No country tax data in profile. Run /finyx:profile to set up your country information."

**Phase 2: Tax Year Staleness Check** (per D-05):
- Get current year: `CURRENT_YEAR=$(date +%Y)`
- Compare against `tax_year` from loaded reference doc frontmatter
- If mismatch: emit warning banner: "Reference doc is from tax year {tax_year}, current year is {CURRENT_YEAR}. Verify rules before acting on this guidance."

**Phase 3: German Investment Tax** (only if Germany active):
- Output banner: `FINYX > TAX: GERMAN INVESTMENT TAX`
- **3.1 Steuerklasse** (DETAX-01): Read `countries.germany.tax_class` from profile. Show explanation of their class from the reference doc. If married, note III/V vs IV/IV optimization. Contextualize with their marginal rate.
- **3.2 Abgeltungssteuer Breakdown** (DETAX-05): Show the flat 25% + Soli calculation. If `countries.germany.church_tax == true`, show church tax rate. Show effective rate. If `countries.germany.marginal_rate < 26.375`, note Guenstigerpruefung eligibility — "Your marginal rate ({rate}%) is below the flat rate (26.375%). You can reclaim the difference via Anlage KAP."
- **3.3 Teilfreistellung** (DETAX-04): Show the fund type table from reference doc. Ask user what fund types they hold (if not obvious from context). Apply correct Teilfreistellung to their scenario.
- **3.4 Sparerpauschbetrag Tracking** (DETAX-02, D-08, D-09, D-10):
  - Determine allowance: if `identity.family_status == "married"` then 2,000 EUR else 1,000 EUR
  - Read `countries.germany.brokers[]` from profile
  - If brokers array is empty: use AskUserQuestion to collect broker data (name, freistellungsauftrag amount, estimated annual capital income). Offer to save to profile.json via Write tool.
  - Calculate: sum of freistellungsauftrag, sum of estimated_annual_income, remaining buffer
  - If sum(freistellungsauftrag) > allowance: WARNING — "Total Freistellungsauftrag ({sum} EUR) exceeds your allowance ({allowance} EUR). This violates section 44a EStG. Reduce allocations immediately."
  - Output: per-broker table in the format shown in 02-RESEARCH.md (Sparerpauschbetrag Report Output Format section)
- **3.5 Vorabpauschale** (DETAX-03): Ask user for their accumulating ETF fund value (or read from context if available). Walk through the formula step by step using current Basiszins from reference doc. Show estimated tax amount. Remind about January cash buffer.

**Phase 4: Brazilian Investment Tax** (only if Brazil active):
- Output banner: `FINYX > TAX: BRAZILIAN INVESTMENT TAX`
- **4.1 IR Filing Guidance** (BRTAX-01): Based on profile, show relevant asset types and their tax treatment from reference doc. Focus on the types the user mentions or is likely to hold based on profile context.
- **4.2 DARF Calculation** (BRTAX-02): If user has stock/FII gains to report, walk through the monthly calculation. Show DARF codes (6015 for normal, 3317 for day-trade). Show deadline formula. Include reminder: "Set a monthly reminder for the last business day of each month."
- **4.3 Come-Cotas** (BRTAX-03): Explain the mechanism. Explicitly state scope: open-end CVM funds only, NOT FIIs or ETFs. Show May/November deduction timing.
- **4.4 FII Dividend Exemption** (BRTAX-04, BRTAX-05): Show base exemption rule (Law 8,668/1993). Show Law 15,270/2025 changes. Include the explicit disclaimer per D-12: "The interaction between Law 15,270/2025 and standard listed FII exemptions has not been fully confirmed by Receita Federal. Verify with a contador before the 2026 tax year."

**Phase 5: Cross-Border DBA Guidance** (only if `identity.cross_border == true`, per D-13):
- Output banner: `FINYX > TAX: CROSS-BORDER (DE-BR) TAX INTERACTION`
- **5.1 Residency Tiebreaker** (D-11): OECD Art. 4 rules — permanent home, habitual abode, nationality, mutual agreement.
- **5.2 Withholding Credit** (D-11): If German resident receiving Brazilian income — claim credit on Anlage AUS. If Brazilian resident receiving German income — credit German withholding in BR filing.
- **5.3 Double-Dip Prevention** (D-11): Do not claim same income as fully exempt in both jurisdictions.
- **5.4 Scope Disclaimer**: "This is basic DBA guidance only. Consult a cross-border tax advisor (Steuerberater with international tax experience or contador with DBA expertise) for complete analysis."
- Note per D-12: "INSS expat treatment and FII exemption edge cases under Law 15,270/2025 are not covered in this version."

**Phase 6: Disclaimer**
- Append the legal disclaimer from the loaded disclaimer.md reference at the end of all advisory output.

**Error handling section:**
- No profile: direct to `/finyx:profile`
- No country data: direct to `/finyx:profile` to complete country sections
- Empty brokers[]: offer interactive collection via AskUserQuestion

**Notes section:**
- The command is conversational — it outputs advisory text, not files
- Vorabpauschale and Sparerpauschbetrag calculations are stateless (per D-08) — computed fresh each run
- Tax-loss harvesting is mentioned briefly in reference docs but detailed guidance is deferred to v2
