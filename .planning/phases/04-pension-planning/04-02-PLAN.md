**`<objective>` section:**
Deliver personalized pension guidance based on the user's financial profile.

This command:
1. Reads `.finyx/profile.json` and detects which countries are active
2. Checks reference doc tax year against the current year and warns if stale
3. For Germany: compares Riester vs Ruerup vs bAV based on employment situation, calculates Riester Zulagen, estimates Ruerup tax saving, explains bAV Entgeltumwandlung benefits and tradeoffs
4. For Brazil: guides PGBL vs VGBL decision based on IR regime and 12% income threshold, explains progressive vs regressive regime with time horizon examples (including Law 14.803/24 deferral)
5. For cross-border users (`cross_border: true`): builds combined retirement projection merging DE statutory + private + BR INSS

Output is conversational advisory text — no files are written unless user consents to saving pension profile data. All guidance includes the legal disclaimer.

**`<process>` section — 7 phases:**

**Phase 1: Validation** (identical pattern to tax.md)
- Check `.finyx/profile.json` exists via bash `[ -f .finyx/profile.json ]`
- Read profile and extract: `identity.cross_border`, `identity.family_status`, `identity.children`, `countries.germany.tax_class` (null = DE not active), `countries.germany.gross_income`, `countries.germany.marginal_rate`, `countries.brazil.ir_regime` (null = BR not active), `countries.brazil.gross_income`, and all `pension.*` fields.
- Determine active countries: Germany active if `tax_class != null`, Brazil active if `ir_regime != null`, cross-border if `identity.cross_border == true`.
- If neither active: show error and stop (same error pattern as tax.md).

**Phase 2: Tax Year Staleness Check** (identical pattern to tax.md)
- Bash: `CURRENT_YEAR=$(date +%Y)` then compare to `tax_year: 2025` in loaded pension reference docs.
- If mismatch, show FINYX PENSION: STALENESS WARNING banner (same format as tax.md but with "PENSION" in header).

**Phase 3: German Pension** (execute only if Germany active) per D-03

Banner: `FINYX > PENSION: GERMAN PENSION PLANNING`

**3.1 Employment Status Assessment**
Read `countries.germany.tax_class` and `identity.family_status` as proxies for employment status. If tax_class is I, II, III, IV, V -> employed (unmittelbar foerderberechtigt for Riester). If the user appears self-employed (this cannot be determined from current profile fields), note it and use AskUserQuestion: "Are you employed (paying mandatory GRV contributions) or self-employed?" This determines Riester eligibility.

**3.2 Riester Zulagen Calculation** (per PENSION-02, D-10)
- Read `identity.children` from profile. If children > 0, use AskUserQuestion to collect birth years (needed for pre/post 2008 Kinderzulage distinction): "You have [N] children in your profile. To calculate your Riester Kinderzulage, I need their birth years. Please list the birth year for each child."
- Calculate: Grundzulage = 175 EUR. Kinderzulage = 300 EUR per child born 2008 or later + 185 EUR per child born before 2008. Total Zulagen = Grundzulage + sum(Kinderzulagen).
- Minimum own contribution: `max(0.04 * countries.germany.gross_income - total_zulagen, 60)` EUR/year.
- Maximum eligible: 2,100 EUR/year (own + Zulagen).
- Show Guenstigerpruefung note: Finanzamt auto-applies whichever is larger.
- Offer to save children birth years to profile if user wants (via Write to .finyx/profile.json, adding `identity.children_birth_years` array).

**3.3 Ruerup Sonderausgabenabzug Estimate** (per PENSION-03)
- Use verified 2025 Hoechstbeitrag: 29,344 EUR single / 58,688 EUR married (NOT 27,566 from D-11 which was the 2024 value).
- Determine single/married from `identity.family_status`.
- Note that GRV contributions count toward Hoechstbeitrag. For employed users, estimate GRV share: approx 18.6% of gross income up to BBG (96,600), split 50/50 employer/employee. Show remaining Ruerup space.
- Calculate: `tax_saving = min(user_stated_contribution_or_max, remaining_ruerup_space) * countries.germany.marginal_rate`.
- If user doesn't state a contribution amount, show the maximum possible tax saving: `hoechstbeitrag * marginal_rate`.

**3.4 bAV Guidance** (per PENSION-01)
- Show: tax-free limit 7,728 EUR/year (8% BBG), SV-free limit 3,864 EUR/year (4% BBG).
- Mandatory 15% employer Zuschuss on Entgeltumwandlung.
- Show tax+SV saving formula: `tax_saving = min(redirect, 7728) * marginal_rate`, `sv_saving = min(redirect, 3864) * 0.207`.
- ALWAYS mention GRV tradeoff: lower gross = fewer Rentenpunkte.
- Generic framing: "Ask your HR/employer about available bAV vehicles."

**3.5 Comparison Summary** (per PENSION-01)
- Render the Riester vs Ruerup vs bAV comparison matrix from the reference doc.
- Add personalized recommendation based on user's situation:
  - Has children + employed -> Riester likely best (Kinderzulagen)
  - High earner + no children + employed -> bAV or Ruerup likely better
  - Self-employed -> Ruerup (primary vehicle)
  - Employed + employer offers bAV match -> bAV first, then Riester/Ruerup

**Phase 4: Brazilian Pension** (execute only if Brazil active) per D-03

Banner: `FINYX > PENSION: BRAZILIAN PENSION PLANNING`

**4.1 PGBL vs VGBL Decision** (per PENSION-04, D-12)
- Read `countries.brazil.ir_regime` from profile.
- If `ir_regime == "completa"`: check INSS contribution status. Use AskUserQuestion if `pension.br_inss_status` is null: "Do you have active INSS or statutory pension contributions? (Required for PGBL deduction)".
- If both conditions met (completa + INSS active): recommend PGBL up to 12% of `countries.brazil.gross_income`. Show: `max_deductible = 0.12 * gross_income`, `tax_saving = max_deductible * marginal_ir_rate`.
- If simplificada or no INSS: recommend VGBL. Explain: VGBL taxes gains only at withdrawal, no current-year deduction.
- Show clear decision tree output.

**4.2 Progressive vs Regressive Regime** (per PENSION-05)
- Show full regressive table (35% to 10% by contribution age per tranche).
- Show progressive table (0% to 27.5% standard IR table).
- CRITICAL: Include Law 14.803/24 note — regime choice can now be DEFERRED to withdrawal time. Recommend deferring unless user has clear >10yr horizon.
- Show example scenarios: 1) R$500k withdrawal after 12 years at regressive = 10%, 2) R$50k/year withdrawal at progressive with low total income = potentially 0-15%.

**4.3 INSS Status Collection** (for cross-country projection, per D-06)
- If `pension.br_inss_status` is null, use AskUserQuestion: "What is your INSS (Brazilian social security) status? Options: (1) Active — still contributing, (2) Suspended — no current contributions but prior contributions remain, (3) Totalization — intend to use DE-BR bilateral agreement to combine contribution periods."
- Explain what each status means for projection.
- Offer to save to profile.json pension block via Write.

**Phase 5: Cross-Country Projection** (per PENSION-06, D-02 — execute ONLY if `identity.cross_border == true`)

Banner: `FINYX > PENSION: CROSS-COUNTRY RETIREMENT PROJECTION`

**5.1 Collect Missing Inputs**
For each null pension field, use AskUserQuestion:
- `pension.target_retirement_age` if null: "At what age do you plan to retire?"
- `pension.de_rentenpunkte` if null: "How many Rentenpunkte (Entgeltpunkte) do you have? Check your annual Renteninformation from Deutsche Rentenversicherung. Enter a number or 'skip' if unknown."
- DE private pension monthly estimate: "Do you have any private pension plans in Germany (Riester, Ruerup, bAV)? If yes, what is your estimated monthly payout at retirement? Enter amount or 'skip'."
- BR INSS monthly benefit estimate (if br_inss_status != null and != "suspended"): "What is your estimated monthly INSS benefit? Enter amount in BRL or 'skip'."
- BR private pension estimate: "Do you have PGBL/VGBL plans? If yes, what is the current accumulated balance? Enter amount in BRL or 'skip'."
- Offer to save all collected pension fields to profile.json.

**5.2 Build Projection Table** (per D-04 inflation-adjusted)
- Calculate years to retirement from `target_retirement_age` and current age (derive from profile if available, otherwise ask).
- Use real rates: `pension.expected_real_return_de` (default 1.5%) and `pension.expected_real_return_br` (default 2.0%).
- Show inflation-adjusted projection:

```
CROSS-COUNTRY RETIREMENT PROJECTION (real terms, [year] EUR)
---
DE Statutory Pension:    [rentenpunkte * 40.79] EUR/month
DE Private Pension:      [user estimate, inflation-adjusted] EUR/month
BR INSS Benefit:         [user estimate] BRL/month (self-reported)
BR Private Pension:      [balance * (1+real_rate)^years / annuity_factor] BRL/month est.
---
Combined Monthly (est.): [sum in EUR, using approximate EUR/BRL]
```

Note: EUR/BRL conversion is approximate. Flag it.

**5.3 D-07 Disclaimer** (MANDATORY, verbatim per D-07):
> Cross-border pension entitlements require verification by a Brazilian social security lawyer (advogado previdenciario). This projection is an illustrative estimate only — actual benefits depend on contribution history, treaty application, and regulatory changes in both countries.

**Phase 6: Disclaimer**
Append the full legal disclaimer from loaded disclaimer.md (same pattern as tax.md Phase 6).

**Phase 7: Save Offer**
If any pension fields were collected via AskUserQuestion during this session, offer to save all of them to `.finyx/profile.json` pension block in a single Write operation.
