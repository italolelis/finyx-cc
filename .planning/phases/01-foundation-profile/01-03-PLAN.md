    **Frontmatter:**
    ```yaml
    ---
    name: finyx:profile
    description: Complete financial profile interview to unlock all Finyx advisors

    **Objective section:** Explain that this creates the financial profile that gates all specialist commands. Creates `.finyx/profile.json`, `.finyx/STATE.md`, and project directories.

    ```
    @~/.claude/finyx/references/disclaimer.md

    **Process phases:**

    **Phase 1: Setup Check**
    ```bash
    [ -f .finyx/profile.json ] && echo "ERROR: Profile already exists. Use /finyx:status to see current state or delete .finyx/ to start fresh." && exit 1
    ```

    **Phase 2: Group 1 -- Residency + Nationality**
    Display banner:
    ```
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
     FINYX > FINANCIAL PROFILE

    ```

    Use AskUserQuestion:
    - header: "Residence Country"
    - question: "Where do you currently live?"
    - options: "Germany", "Brazil", "Other"

    Use AskUserQuestion:
    - header: "Nationality"
    - question: "What is your nationality?"
    - options: "German", "Brazilian", "Other"

    Ask inline: "Do you earn income in any other country besides your country of residence? If yes, which countries?"

    **After group 1, compute cross_border (per D-08):**
    ```
    cross_border = false
    if residence_country != nationality_country:
      cross_border = true
    if income_countries has more than one entry:
      cross_border = true

    If cross_border:
      Display: "Cross-border situation detected (residence: [X], nationality: [Y]). We'll collect details for all relevant jurisdictions."
    ```

    **Phase 3: Group 2 -- Income + Tax (conditional on countries)**

    If Germany is relevant (residence OR income country):
    - AskUserQuestion for tax class: "I (Single)", "III/V (Married, one high earner)", "IV/IV (Married, equal earners)"
    - AskUserQuestion for church tax: "No" (default), "Yes (8-9% additional)"
    - Ask inline for annual gross income in Germany
    - Calculate marginal rate using the formula from tax-rules.md reference:
      ```
      If total income > 277,826 EUR (married) or 68,480 EUR (single): baseRate = 42%
      marginalRate = baseRate + (baseRate * 5.5%)  # Soli
      If churchTax: marginalRate += baseRate * 8%
      ```
    - Display calculated rate, ask for confirmation

    If Brazil is relevant (residence OR income country):
    - AskUserQuestion for IR regime: "Simplificado (20% standard deduction)", "Completo (itemized deductions)"
    - Ask inline for annual gross income in Brazil
    - Ask inline for CPF number (optional, for reference only)

    **Phase 4: Group 3 -- Goals + Risk Tolerance**

    Use AskUserQuestion:
    - header: "Family Status"
    - question: "What is your family status?"
    - options: "Single", "Married", "Registered Partnership", "Divorced/Separated"

    Ask inline: "How many children do you have (if any)?"

    Use AskUserQuestion:
    - header: "Risk Tolerance"
    - question: "How would you describe your risk tolerance?"
    - options:
      - "Conservative -- Preserve capital, minimal risk"
      - "Moderate -- Balanced growth and safety"
      - "Aggressive -- Maximum growth, accept volatility"

    Use AskUserQuestion:
    - header: "Investment Horizon"
    - question: "What is your primary investment time horizon?"
    - options:
      - "Short-term (1-3 years)"
      - "Medium-term (3-10 years)"
      - "Long-term (10+ years)"

    Ask inline: "What are your primary financial goals? (e.g., real estate investment, ETF portfolio, retirement savings, tax optimization)"

    Use AskUserQuestion:
    - header: "Liquid Assets"
    - question: "What range best describes your total liquid assets available for investment?"
    - options:
      - "Under 25,000 EUR"
      - "25,000 - 100,000 EUR"
      - "100,000 - 500,000 EUR"
      - "Over 500,000 EUR"

    Ask inline for exact liquid assets amount and monthly financial commitments.

    **Phase 5: Create Project Files**

    Create directory structure:
    ```bash
    mkdir -p .finyx/research/market
    mkdir -p .finyx/research/locations
    mkdir -p .finyx/analysis
    mkdir -p .finyx/output
    mkdir -p properties
    ```

    Write `.finyx/profile.json` using the template schema, populating ALL fields from interview answers:
    - `identity.*` from group 1
    - `countries.*` from group 2 (only populate countries that are relevant)
    - `goals.*` from group 3
    - `investor.*` populated for backward compat:
      - `investor.country` = residence_country
      - `investor.taxClass` = germany.tax_class (if DE) or ""
      - `investor.churchTax` = germany.church_tax (if DE) or false
      - `investor.marginalRate` = germany.marginal_rate (if DE) or 0
      - `investor.income.primary` = primary income
      - `investor.liquidAssets` = from interview
      - `investor.monthlyCommitments` = from interview
    - `strategy.*` and `criteria.*` use template defaults (RE-specific, set via future commands)
    - `assumptions.*` use template defaults

    Write `.finyx/STATE.md` from state template, updated for Finyx branding.

    Write `FINYX.md` project summary in project root.

    **Phase 6: Completion**

    Display summary banner with FINYX branding showing profile summary, detected cross-border status, and next steps (run `/finyx:scout` for RE, other specialist commands coming in future phases).

    Append the legal disclaimer at the end of the output.

    **Error handling section:**
    - Profile exists: direct to `/finyx:status`
    - Invalid income: prompt for clarification
    - Country not supported: inform user, only DE and BR have full support in v1
