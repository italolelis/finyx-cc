**Process section with these phases:**

**Phase 1: Validation and Completeness Gate (INFRA-01)**
- Bash check: `[ -f .finyx/profile.json ] || { echo "ERROR: No financial profile found. Run /finyx:profile first."; exit 1; }`
- Read profile.json (already in execution_context)
- Check required fields per agent (per research Pattern 3):
  - Allocation agent: `investor.income.total` or `countries.germany.gross_income`, `investor.monthlyCommitments`, `investor.liquidAssets`
  - Tax scoring agent: `countries.germany.gross_income > 0` OR (`identity.cross_border == true` AND `countries.brazil.gross_income > 0`), `countries.germany.tax_class` (for DE), `countries.brazil.ir_regime` (for BR, only if cross_border)
  - Projection agent: `investor.liquidAssets`, `investor.monthlyCommitments`, `investor.income.total` or `countries.germany.gross_income`
- Cross-border gate: if `identity.cross_border == true`, also check `countries.brazil.gross_income > 0` before requiring Brazil fields (Pitfall 6)
- If ANY required field is missing/null/zero: emit completeness report using the FINYX INSIGHTS: PROFILE INCOMPLETE banner format (listing each missing field with description), then STOP — do NOT spawn agents
- If all fields present: proceed to Phase 2

**Phase 2: Legal Disclaimer (INFRA-03, D-07)**
- Emit the FINYX INSIGHTS banner header
- IMMEDIATELY emit the legal disclaimer from loaded `disclaimer.md` BEFORE any advisory content (per D-07, Pitfall 2)
- Use the standard disclaimer banner format from tax.md

**Phase 3: Agent Spawning (D-05)**
- Check for existing allocation mapping: `[ -f .finyx/insights-config.json ]` — if exists, read it and note that mapping confirmation can be skipped
- Spawn ALL 3 agents in PARALLEL via the Task tool (per D-05):
  - Task 1: finyx-allocation-agent — provide the income/expenses/goals/investments profile slice. If `.finyx/insights-config.json` exists with `allocation_mapping`, include it in the agent prompt so it skips the AskUserQuestion confirmation flow.
  - Task 2: finyx-tax-scoring-agent — provide the countries/tax/investments profile slice
  - Task 3: finyx-projection-agent — provide the assets/liabilities/goals profile slice
- Collect XML-tagged outputs: `<allocation_result>`, `<tax_score_result>`, `<projection_result>`

**Phase 4: Allocation Mapping Persistence (D-08)**
- If `<allocation_result>` contains an `<allocation_mapping_confirmed>` block (first-run scenario):
  - Extract the mapping JSON
  - Write it to `.finyx/insights-config.json` under the `allocation_mapping` key
  - Ensure `.finyx/` directory exists: `[ -d .finyx ] || mkdir -p .finyx`
- If no `<allocation_mapping_confirmed>` block: skip (mapping was already persisted from a prior run)

**Phase 5: Report Synthesis (D-01, D-02, REC-01, REC-02)**
Assemble the unified report in the EXACT section order per D-01:

**Section 1 — Summary:**
- Net worth snapshot from `<projection_result>` (assets vs liabilities, conservative/base ranges)
- Key metrics: savings rate, investment rate, overall efficiency indicators

**Section 2 — Health Dashboard (D-02):**
- Single unified traffic-light table with Country column — NOT separate per-country blocks
- Format:
```
| Dimension | Country | Status | Gap | How to Close |
|-----------|---------|--------|-----|--------------|
```
- Populate from agent outputs: allocation dimensions from `<allocation_result>`, tax dimensions from `<tax_score_result>`, projection indicators from `<projection_result>`
- Status values: [GREEN], [YELLOW], [RED] — read directly from agent outputs (agents already produce scored output)

**Section 3 — Top-5 Recommendations (REC-01, D-06):**
- Collect ALL gap amounts from agent outputs (every [RED] or [YELLOW] dimension has an explicit EUR/BRL gap)
- For BRL gaps: convert to EUR using a stated assumption ("assuming BRL/EUR = 0.18 for comparison only")
- Sort by annual EUR impact descending, take top 5
- Format:
```
| # | Action | Est. Annual Impact | Dimension |
|---|--------|--------------------|-----------|
```

**Cross-Advisor Intelligence (REC-02, D-03):**
Include a `<cross_advisor_links>` section in the command prompt that enumerates these patterns for Claude to detect from collected agent outputs:
- Unused Sparerpauschbetrag + low investment rate = double gap (TAX-01 red/yellow AND ALLOC-01 red)
- No pension contributions + high marginal rate = missed Ruerup deduction (marginal_rate > 42% AND no pension data)
- Emergency fund shortfall + high investment rate = over-allocated risk (ALLOC-02 red AND high investment allocation)
- PGBL not used + completo regime = missed BR deduction (TAX-04 red AND ir_regime == "completo")
- Claude may also surface novel combinations beyond these examples
- Any cross-advisor link that fires should appear as a recommendation in Section 3, ranked by estimated annual EUR/BRL impact

**Section 4 — Detail:**
- 4.1 Allocation detail: render the full allocation breakdown from `<allocation_result>`
- 4.2 Tax efficiency detail: render the full per-country scoring from `<tax_score_result>`
- 4.3 Projections and goals: render the full net worth + goal pace from `<projection_result>`

**error_handling section:**
- No profile found: standard error per tax.md pattern
- Profile incomplete: handled in Phase 1 completeness gate
- Agent returns data gap notes: include them inline in the detail section with a [DATA GAP] label

**notes section:**
- This command writes ONLY to `.finyx/insights-config.json` (allocation mapping persistence) — never to profile.json
- Agents are stateless and read-only — the orchestrator handles all Write operations
- DE and BR dimensions are NEVER combined into a single score (per project-wide rule)
- BRL/EUR conversion rate is a stated assumption for ranking only — not a live rate
- The disclaimer MUST appear before any financial content (not at the end like tax.md/invest.md)
