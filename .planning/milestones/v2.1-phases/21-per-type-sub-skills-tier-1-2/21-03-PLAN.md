**File structure:**
1. `# Kfz Insurance Sub-skill` heading
2. HTML comment: `<!-- Sub-skill loaded by router SKILL.md. All CLAUDE_SKILL_DIR paths resolve to skills/insurance/ -->`
3. `<objective>` — Deliver personalized Kfz insurance analysis covering all 3 coverage tiers (Haftpflicht, Teilkasko, Vollkasko), SF-Klasse impact, vehicle-specific recommendations, cancellation tracking, and criteria-based market research. Read-only, writes NO files.

4. `<process>` with 7 phases:

**Phase 0: Preferences (EXTENDED — per CONTEXT.md locked decisions)**

This phase collects 6 data points via AskUserQuestion. Collect in logical groupings (2-3 questions per round if possible).

Q1 — Current coverage tier (singleSelect):
"What is your current Kfz insurance coverage level?"
- Haftpflicht only (third-party liability)
- Haftpflicht + Teilkasko (partial comprehensive)
- Vollkasko (full comprehensive)
- Not currently insured / new vehicle
- Not sure

Q2 — SF-Klasse (text input with explanation):
"What is your SF-Klasse (Schadenfreiheitsklasse)?
This is your no-claims bonus class — it directly affects your premium.
0 = new driver / no claims history, 35 = 35+ claim-free years.
A single at-fault accident typically drops you 4-6 classes.
Find it on your current policy document (Versicherungsschein)."
Accept numeric input 0-35. If user doesn't know: use "unknown" and note in Phase 3 that premium estimates won't be personalized.

Q3 — Vehicle age (singleSelect):
"How old is your vehicle?"
- 0-3 years (new/nearly new)
- 3-7 years
- 7-12 years
- 12+ years

Q4 — Annual km (singleSelect):
"How many kilometers do you drive per year?"
- Under 10,000 km
- 10,000–20,000 km
- 20,000–30,000 km
- Over 30,000 km

Q5 — Garage (singleSelect):
"Is your vehicle parked in a garage overnight?"
- Yes — private garage
- No — street parking

Q6 — Financed (singleSelect):
"Is your vehicle financed or leased?"
- Yes — financed/leased
- No — fully owned

Store all as `vehicle_data` object:
```
vehicle_data = {
  current_tier: [selected],
  sf_klasse: [number or "unknown"],
  vehicle_age: [selected bracket],
  annual_km: [selected bracket],
  garage: [yes/no],
  financed: [yes/no]
}
```

**Phase 3: Coverage Benchmark Comparison** — Read from `${CLAUDE_SKILL_DIR}/references/germany/kfz.md`.

**3a: 3-Tier Comparison Table** (ALWAYS show, this is the core output):
```
| Criterion | Haftpflicht | Teilkasko | Vollkasko |
|-----------|-------------|-----------|-----------|
| Third-party damage (Personen/Sachschäden) | MANDATORY (§1 PflVG) | Included | Included |
| Theft (Diebstahl) | Not covered | COVERED | COVERED |
| Weather/natural events (Elementarschäden) | Not covered | COVERED | COVERED |
| Glass breakage (Glasbruch) | Not covered | COVERED | COVERED |
| Animal collision (Wildunfall/Tierbiss) | Not covered | COVERED | COVERED |
| Own-fault accident damage | Not covered | Not covered | COVERED |
| Vandalism (Vandalismus) | Not covered | Not covered | COVERED |
| Typical monthly premium range | €X–Y | €X–Y | €X–Y |
```
Read premium ranges from reference doc — do NOT hardcode. Mark with "[from reference doc]" if ranges need updating.

**3b: When-to-Upgrade Guidance** (from kfz.md Coverage Benchmarks):
Based on vehicle_data.vehicle_age collected in Phase 0:
- 0-3 years → "Vollkasko strongly recommended — vehicle value is high, repair costs are expensive"
- 3-7 years → "Vollkasko or Teilkasko — consider vehicle's current market value vs premium difference"
- 7-12 years → "Teilkasko usually sufficient — Vollkasko premium may exceed vehicle's damage potential"
- 12+ years → "Haftpflicht or Teilkasko — Vollkasko rarely cost-effective for older vehicles"

If `vehicle_data.financed == "Yes"`: "⚠️ Financed/leased vehicles typically REQUIRE Vollkasko — check your finance agreement."

**3c: SF-Klasse Impact** (if sf_klasse is known):
Show impact explanation: "Your SF-Klasse {N} translates to approximately {discount}% premium discount. At-fault accidents typically result in a downgrade of 4-6 classes."
Read SF-Klasse discount table from reference doc if available. If not, show general guidance only.

**3d: Current Coverage Assessment** (if existing_policy_found):
Compare current tier from `coverage_components` against recommendation from 3b:
- If user has Haftpflicht only but vehicle is 0-3 years: "Your vehicle age suggests Vollkasko — you currently have Haftpflicht only."
- If user has Vollkasko but vehicle is 12+ years: "Your vehicle age suggests Teilkasko or Haftpflicht may be more cost-effective."

**Phase 4: Cancellation Deadline Check** — SPECIAL RULE for Kfz:
- Standard Kündigungsfrist is 1 month — deadline is typically 30.11 (November 30) for 01.01 (January 1) renewal
- Most Kfz policies renew on January 1 regardless of start date (Versicherungsjahr = Kalenderjahr)
- Sonderkündigungsrecht triggers: after a claim (insurer or policyholder), after SF-Klasse reclassification, after premium increase without coverage change

Same banner patterns. If `start_date` and `kuendigungsfrist_months` present: compute deadline. Otherwise show "UNKNOWN" fallback. Mention the 30.11 deadline explicitly as most Kfz policies follow this pattern.

```
<user_preferences>

```
Render the `<insurance_research_result>` output.

**Phase 6: Recommendation** — Synthesize all data:
- Tier recommendation based on vehicle age + financing + current coverage
- SF-Klasse impact awareness
- Coverage gaps (e.g., Haftpflicht only for 2-year-old financed car = major gap)
- Cancellation window information
- Market context from research agent
- Concrete next steps with EUR estimates where possible

If user is overinsured (Vollkasko on 15-year-old unfinanced car): suggest downgrading with estimated savings.
If user is underinsured (Haftpflicht only on new financed car): flag urgently.
